Flag 2 – Exploração de plugin WordPress

Após enumeração de diretórios no servidor web foi identificado um diretório:

/wordpress

Foi executado um scan utilizando WPScan, que identificou o plugin vulnerável:

simply-poll
version: 1.4.1

Na base de dados do Exploit-DB foi identificado que esse plugin possui vulnerabilidades:

XSS

SQL Injection

Foi utilizada a exploração de SQL Injection.

Comando sugerido no Exploit-DB
sqlmap -u "http://example.com/wp-admin/admin-ajax.php"
--data="action=spAjaxResults&pollid=2"
--dump -T wp_users -D wordpress
--threads=10 --random-agent --dbms=mysql --level=5 --risk=3
Comando utilizado no ataque
sqlmap -u "http://3.219.234.210/wp-admin/admin-ajax.php"

Flag 4 – Escalação de privilégios

Dentro do SSH foi encontrado o arquivo:

Politicas_de_senhas

O arquivo mencionava um usuário chamado carlos, chefe da TI.

Foi analisado o histórico de comandos do usuário:

cat .bash_history

No histórico foi possível observar:

criação de um script chamado backup.sh

envio de arquivos utilizando nc

criação de um arquivo:

Trabalho_Estagiario.zip

Também foi identificado um diretório:

flag4

Porém sem permissão de acesso.

Análise do script backup.sh

Dentro da home do usuário foi encontrado o script:

backup.sh

Conteúdo:

#!/bin/bash

cp -R /root /var/backups
cp -R /etc /var/backups/root
zip -P estagiario -r Trabalho_Estagiario.zip /var/backups/root && mv Trabalho_Estagiario.zip /var/backups
sudo chmod -R ugo+rw /var/backups/Trabalho_Estagiario.zip
sudo chmod -R estagiario /var/backups/Trabalho_Estagiario.zip
rm -r /var/backups/root

Observações importantes:

O ZIP possui senha estagiario

O arquivo recebe permissões amplas

Contém backup de:

/root
/etc
Extração do backup

Foi acessado o diretório:

/var/backups

Localizando o arquivo:

Trabalho_Estagiario.zip

Extração do arquivo:

unzip Trabalho_Estagiario.zip

Senha:

estagiario

Dentro do ZIP foi encontrada uma cópia de:

/etc
/root

Isso permitiu acessar arquivos sensíveis como:

/etc/shadow

e também a pasta:

/root/.ssh

Dentro dela havia uma chave RSA privada.

ssh root@127.0.0.1 -i rsa
